1.7 Installation

This particular text can be seen by running ‘make help-install’.

After the building process (see Build), comes the last step: the installation of the package. The primary used for this is ‘all-install[a][f][l]’. Therefore, to install all collection and standalone formats, in all available languages, invoke ‘make all-install’. As explained in Goal suffixes, you can further specify the installation by using any combination of aggregate, format and language suffixes. Notice that invoking the installation of an aggregate type, output format or translation language not already built will automatically trigger its build before installation.

The GNU website philosophy section essays and the homepage of GNU philosophy package cannot be installed, but are available in the directories (relative to the build top directory) essays/src/www/www-gnu and essays/src/www/homepage.

The installation puts the built essays documents into the directory 

docdir/language/aggregate/format

where docdir, language, aggregate and format, are the documentation directory, which can be specified with the script configure option --docdir and defaults to /usr/local/share/doc/philosophy, the translation language, ‘en’ for the original essays, aggregate type and output format.

Naturally, there are a goal to uninstall the package almost analogous to its installation counterpart: ‘all-uninstall[a][f][l]’. Obviously, it will not trigger the build and installation before the removal of the selected set of essays.
